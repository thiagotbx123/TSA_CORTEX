## Decisões Tomadas
| Decisão | Contexto | Alternativas Consideradas |
|---------|----------|---------------------------|
| "Product Readiness" como sugestão de categoria | Paralelo com Gong (Revenue Intelligence) | Product Orchestration, GTM Infrastructure, Product Experience |
| Service model como diferencial | TSA + DE embedded é moat difícil de copiar | Focar só em tecnologia |
| DE deve estar em todas as fases com data work | Thais/Yasmin são time dedicado, não "if available" | Manter DE só na Phase 5 |
| Modelo de dados: FDE define → DE gera → FDE valida | Clarificar ownership de datasets | TSA faz tudo, DE só valida |

## Conhecimentos Adquiridos
- **Category Definition:** TestBox não é "demo automation" - é uma categoria nova que combina product readiness + service model
- **Delivery Process:** DE precisa ser incorporado desde Phase 1 (resource allocation) até Phase 5 (validation)
- **DE Gate:** Performance Excellence define DE Gate Compliance como behavior core - RFC Phases deve refletir isso
- **Terminologia:** TSA → FDE (Forward Deployed Engineer) é a nova nomenclatura
